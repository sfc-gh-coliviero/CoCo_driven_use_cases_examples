1 Cos'è Cortex Code?

Cortex Code è l'assistente di programmazione con IA integrato in Snowsight, l'ambiente web di Snowflake. Permette di generare codice SQL, Python e Streamlit direttamente dal linguaggio naturale, accelerando lo sviluppo di soluzioni analitiche.

Funzionalità principali

  • Generazione codice SQL — CREATE TABLE, INSERT, viste, procedure, funzioni
  • Cortex AI/ML — ML.CLASSIFICATION, ML.FORECAST, CORTEX.COMPLETE, CORTEX.SENTIMENT, Cortex Search
  • Feature Store — Entity, FeatureView, get_features()
  • Model Registry — log_model(), get_model(), versionamento
  • Streamlit — Generazione di dashboard interattive
  • Tasks & Pipeline — Automazione e orchestrazione

Importante: Cortex Code genera codice che viene eseguito nel tuo account Snowflake. Tutto rimane nel tuo ambiente — senza dipendenze esterne, senza inviare dati all'esterno.

2 Prerequisiti

Prima di iniziare, assicurati di avere quanto segue configurato:

Account Snowflake

Importante: Richiedi al tuo referente Snowflake la creazione degli account necessari per il tuo team. In questo modo assicuriamo che siano supportate tutte le funzioni di Cortex Code e Cortex AI: ML.CLASSIFICATION, ML.FORECAST, CORTEX.COMPLETE, CORTEX.SENTIMENT, Cortex Search e Feature Store.

Accedere a Cortex Code in Snowsight

Una volta nel tuo account, nella parte destra dello schermo appare il pannello dell'assistente Cortex Code. È il tuo compagno di programmazione IA per eseguire tutti i prompt di questo catalogo:

Pronto: Se vedi il pannello di Cortex Code con il messaggio «How can I help?» a destra, puoi iniziare a copiare e incollare i prompt di qualsiasi caso d'uso.

Account e permessi

  • Account Snowflake con accesso a Snowsight
  • Ruolo con permessi CREATE DATABASE, CREATE SCHEMA, CREATE TABLE
  • Warehouse attivo (consigliato: size XS per sviluppo, S per produzione, meglio ancora usiamo le adaptive warehouse che automaticamente usano la migliore dimensione per il job assegnato)
  • Cortex Code abilitato nell'account (disponibile in tutte le edizioni)

Per funzioni avanzate

  • Cortex AI/ML: Richiede che la regione supporti Cortex Functions (AWS us-west-2, us-east-1, eu-west-1, ecc.)
  • Feature Store: Pacchetto snowflake-ml-python disponibile
  • Model Registry: Permessi per creare modelli nello schema
  • Streamlit: Permesso CREATE STREAMLIT nello schema

Nota sulle regioni: Alcune funzioni di Cortex AI (come ML.CLASSIFICATION o CORTEX.COMPLETE) richiedono regioni specifiche. Consulta la documentazione sulla disponibilità per verificare la tua regione.

3 Aprire Cortex Code in Snowsight

Cortex Code è integrato in diversi posti di Snowsight. Ecco i modi per accedervi:

Opzione A — Da Workspaces

  1. Entra in Snowsight → Projects → Workspaces
  2. Crea o apri un workspace esistente
  3. Fai clic sull'icona di Cortex (✨) nella barra laterale destra, oppure usa la scorciatoia Cmd+Shift+Space (Mac) / Ctrl+Shift+Space (Windows)
  4. Si aprirà il pannello di Cortex Code alla destra dell'editor

Suggerimento: Il pannello di Cortex Code mantiene il contesto del tuo workspace. Se hai tabelle referenziate nell'editor, Cortex Code le userà come contesto per generare codice più preciso.

Struttura di ogni caso d'uso

Facendo clic su «Vedi Prompt» in qualsiasi scheda caso d'uso, vedrai:

  • Passo 1 — Configurare ambiente: Crea il database, lo schema e il warehouse. È sempre il primo prompt.
  • Passi 2-4 — Creare dati: Genera tabelle con dati sintetici rappresentativi del dominio.
  • Passi 5-7 — Analizzare e modellare: Feature, modelli ML, funzioni Cortex AI, Feature Store o Model Registry.
  • Passo 8-9 — Dashboard: Codice Streamlit per visualizzazione interattiva.
  • Ultimo passo — Pipeline: Task per automazione e operatività continua.

Sequenzialità: I prompt sono progettati per essere eseguiti in ordine. Ogni passo dipende dagli oggetti creati nei passi precedenti. Non saltare i passaggi.

6 Eseguire un Caso d'Uso Passo a Passo

Vediamo il processo completo con un esempio pratico:

6.1 — Selezionare settore e caso d'uso

  1. Vai al catalogo dei settori
  2. Scegli un settore (ad esempio, Banca)
  3. Naviga le schede o usa il campo di ricerca per trovare un caso d'uso
  4. Leggi la sezione «Problema» per comprendere il problema di business

6.2 — Leggere la Guida

  1. Fai clic su «Vedi Guida»
  2. Leggi le sezioni: Contesto, focus, sfida, obiettivi, funzionalità, dati, stratificazione e istruzioni d'uso
  3. Questo ti darà il quadro concettuale prima di generare codice

6.3 — Eseguire i Prompt

  1. Fai clic su «Vedi Prompt»
  2. In Snowsight, apri un workspace e attiva Cortex Code
  3. Per ogni passo:
    • Fai clic su «Copia» sul prompt
    • Incollalo nel pannello di Cortex Code
    • Cortex Code genererà il codice SQL/Python corrispondente
    • Riesaminarlo prima di eseguire (vedi sezione 7)
    • Fai clic su «Run» o «Apply»
    • Verifica che sia stato eseguito correttamente
    • Passa al prompt successivo
Esempio-- Prompt copiato dal catalogo (Passo 1): "Crea il database BANCA_FRODE_DB con schema ANALYTICS e warehouse BANCA_WH (size S). Includi commenti del progetto di rilevamento frodi transazionali." -- Cortex Code genera: CREATE DATABASE IF NOT EXISTS BANCA_FRODE_DB COMMENT = 'Progetto di rilevamento frodi transazionali con Cortex AI/ML'; CREATE SCHEMA IF NOT EXISTS BANCA_FRODE_DB.ANALYTICS COMMENT = 'Schema principale per analytics delle frodi'; CREATE WAREHOUSE IF NOT EXISTS BANCA_WH WAREHOUSE_SIZE = 'SMALL' AUTO_SUSPEND = 300 AUTO_RESUME = TRUE COMMENT = 'Warehouse per progetto frodi bancarie'; USE DATABASE BANCA_FRODE_DB; USE SCHEMA ANALYTICS; USE WAREHOUSE BANCA_WH;

Risultato atteso: Al termine di tutti i passi avrai: database con tabelle di dati sintetici, modelli ML addestrati, funzioni Cortex AI configurate e un dashboard Streamlit funzionante.

7 Revisionare e Validare il Codice Generato

Cortex Code genera codice di alta qualità, ma devi sempre riesaminarlo prima di eseguirlo. Punti chiave:

Checklist di revisione

  • Nomi degli oggetti: Verifica che database, schema e warehouse corrispondano al tuo ambiente
  • Warehouse size: Regola la dimensione in base al carico (XS per sviluppo, S per produzione, oppure meglio adaptive)
  • Volume dati sintetici: I prompt suggeriscono volumi rappresentativi; regolali secondo necessità
  • Modelli LLM: Verifica che il modello referenziato (es. llama3.1-70b, mistral-large2) sia disponibile nella tua regione
  • Funzioni ML: Conferma che ML.CLASSIFICATION, ML.FORECAST siano disponibili
  • Permessi: Il tuo ruolo deve avere i grant necessari sugli oggetti

Adattare a dati reali

I prompt generano dati sintetici per dimostrazione. Per portare in produzione:

  1. Sostituisci i CREATE TABLE ... INSERT sintetici con CREATE VIEW o SELECT sulle tue tabelle reali
  2. Adatta i nomi delle colonne al tuo modello dati
  3. Modifica soglie e parametri di business
  4. Mantieni la struttura di feature, modelli e dashboard

Dati sintetici: I dati generati sono fittizi e rappresentativi. Non usare in report di business reali. Servono per validare la pipeline end-to-end prima di connettere dati reali.

8 Pubblicare le dashboard Streamlit

Quasi tutti i casi d'uso includono un prompt per generare un dashboard Streamlit. Ecco come fare il deploy:

8.1 — Generare l'app Streamlit

  1. Esegui il prompt «Dashboard» (di solito il penultimo passo)
  2. Cortex Code genererà codice Python con Streamlit
  3. Entra in Snowsight → Projects → Streamlit

8.3 — Verificare e condividere

  1. Controlla che tutti i grafici e le tabelle si carichino correttamente
  2. Se ci sono errori, verifica che le tabelle e le viste esistano (create nei passi precedenti)
  3. Condividi l'app con altri utenti usando «Share»

9 Automatizzare con Tasks e Pipeline

L'ultimo prompt di ogni caso d'uso crea Task di Snowflake per automatizzare la pipeline:

Cosa si automatizza

  • Ingestione dati: Caricamento periodico da sorgenti (giornaliero, settimanale)
  • Aggiornamento feature: Ricalcolare indicatori e metriche
  • Scoring ML: Eseguire modelli su nuovi dati
  • Alert: Notifiche quando si superano le soglie
  • Riaddestramento: Aggiornare modelli periodicamente

Struttura tipica delle Task

SQL-- Task di scoring settimanale CREATE OR REPLACE TASK scoring_settimanale WAREHOUSE = MIO_WH SCHEDULE = 'USING CRON 0 6 * * 1 Europe/Rome' COMMENT = 'Scoring ML settimanale su dati attivi' AS CALL eseguire_scoring(); -- Attivare ALTER TASK scoring_settimanale RESUME;

DAG di Task: I casi d'uso avanzati creano DAG (Directed Acyclic Graph) dove alcune Task dipendono da altre. Cortex Code genera le dipendenze automaticamente con AFTER.

10 Verificare i Costi Sostenuti

Dopo aver eseguito uno o più casi d'uso, utilizza questo prompt in Cortex Code per ottenere un riepilogo consolidato di tutti i crediti e token consumati nella giornata odierna: warehouse, funzioni Cortex AI, modelli ML e Cortex Code in Snowsight.

Prompt — Riepilogo costi del giorno

Copia e incolla questo prompt direttamente in Cortex Code (ricorda di attivare /best-practices prima di eseguirlo):

PromptGenera una query SQL che mostri un riepilogo consolidato di tutti i costi e consumi dell'account Snowflake del giorno corrente (CURRENT_DATE). Includi le seguenti sezioni: 1. CREDITI WAREHOUSE - Fonte: SNOWFLAKE.ACCOUNT_USAGE.WAREHOUSE_METERING_HISTORY - Colonne: warehouse_name, SUM(credits_used) AS crediti_warehouse - Filtra per: DATE(start_time) = CURRENT_DATE() 2. FUNZIONI CORTEX AI (token e crediti) - Fonte: SNOWFLAKE.ACCOUNT_USAGE.CORTEX_FUNCTIONS_USAGE_HISTORY - Colonne: function_name, SUM(tokens_used) AS token_usati, SUM(credits_used) AS crediti_cortex - Filtra per: DATE(start_time) = CURRENT_DATE() 3. FUNZIONI ML SERVERLESS (ML.CLASSIFICATION, ML.FORECAST, ecc.) - Fonte: SNOWFLAKE.ACCOUNT_USAGE.METERING_DAILY_HISTORY - Filtra per: service_type IN ('ML_FUNCTIONS','SNOWPARK_ML') AND DATE(usage_date) = CURRENT_DATE() - Colonne: service_type, credits_used_compute AS crediti_ml 4. CORTEX CODE IN SNOWSIGHT - Fonte: SNOWFLAKE.ACCOUNT_USAGE.METERING_DAILY_HISTORY - Filtra per: service_type = 'CORTEX_CODE' AND DATE(usage_date) = CURRENT_DATE() - Colonne: service_type, credits_used_compute AS crediti_cortex_code 5. RIEPILOGO TOTALE PER TIPO DI SERVIZIO - Unisci tutti i risultati precedenti con UNION ALL - Colonne finali: tipo_servizio, nome, token_usati, crediti_spesi - Ordina per crediti_spesi DESC - Aggiungi una riga TOTALE alla fine con SUM di tutti i crediti Usa CTE per ogni sezione. Se una vista non esiste nell'account, sostituiscila con SNOWFLAKE.ACCOUNT_USAGE.METERING_DAILY_HISTORY filtrando per il service_type equivalente. Aggiungi commenti che spieghino ogni CTE.

Nota: I dati di ACCOUNT_USAGE hanno una latenza fino a 45 minuti. Se hai appena eseguito un caso d'uso e non appaiono dati, attendi qualche minuto e rilancia la query.

Alternativa: Se CORTEX_FUNCTIONS_USAGE_HISTORY non è disponibile nel tuo account, Cortex Code sostituirà automaticamente quella sezione con METERING_DAILY_HISTORY con service_type = 'AI_SERVICES'.

11 Consigli e Best Practice

Sviluppo

  • Usa sempre /best-practices prima di ogni prompt — Attiva la skill affinché Cortex Code generi codice ottimizzato in prestazioni e costi fin dal primo passo
  • Esegui prompt per prompt — Non tentare di eseguire tutti i prompt contemporaneamente; ogni passo valida il precedente
  • Nomina gli oggetti in modo consistente — I prompt suggeriscono nomi descrittivi come BANCA_FRODE_DB
  • Salva il worksheet — Ogni caso d'uso genera parecchio codice; salva frequentemente

Cortex Code

  • Sii specifico nei prompt — I prompt del catalogo sono già dettagliati, ma puoi aggiungere contesto del tuo business
  • Itera sul codice — Se il codice generato non è esattamente ciò che ti serve, chiedi a Cortex Code di adattarlo
  • Usa il contesto del worksheet — Se hai già tabelle nell'editor, Cortex Code le referenzierà automaticamente
  • Combina prompt se necessario — Per utenti avanzati, puoi combinare 2-3 passi in un solo prompt più lungo

Produzione

  • Sostituisci dati sintetici con reali — La pipeline funziona allo stesso modo, cambi solo la sorgente dati
  • Configura Resource Monitor — Controlla il consumo di crediti del warehouse
  • Revisiona i permessi — Usa ruoli specifici per ogni progetto (data_scientist, analyst, ecc.)
  • Monitora le Task — Controlla lo storico esecuzioni in Activite → Task History
  • Versiona i modelli — I casi con Model Registry includono già il versionamento; usalo per A/B testing

12 Domande Frequenti

Devo conoscere SQL per usare i prompt?
Non è indispensabile, ma aiuta. Cortex Code genera l'SQL per te, ma comprendere i concetti base (CREATE TABLE, SELECT, JOIN) ti permetterà di revisionare e adattare il codice con sicurezza.
I dati sintetici consumano molti crediti?
Minimo. La generazione di dati sintetici con INSERT consuma pochissimi crediti (secondi di warehouse S). Il consumo maggiore deriva dall'addestramento ML e dalle funzioni Cortex, che dipendono dal volume dei dati.
Posso usare i prompt con i miei dati reali fin dall'inizio?
Sì. Sostituisci i passi di «Creare dati» con riferimenti alle tue tabelle esistenti. Assicurati che i nomi delle colonne corrispondano a quanto atteso dai passi successivi, oppure chiedi a Cortex Code di adattare il codice.
Cosa faccio se un prompt genera un errore?
Le cause più comuni: (1) manca l'esecuzione di un passo precedente che crea la tabella referenziata, (2) il warehouse non è attivo, (3) la funzione Cortex non è disponibile nella tua regione. Leggi il messaggio di errore e chiedi a Cortex Code di correggerlo: «Questo codice dà errore X, correggilo».
Quanto tempo ci vuole per completare un caso d'uso?
Tra 8 e 15 minuti per eseguire tutti i prompt con dati sintetici. Il tempo indicato in ogni scheda è una stima realistica che includella revisione del codice.
Posso combinare casi d'uso di settori diversi?
Assolutamente. Molti pattern sono trasversali: un Feature Store cliente funziona allo stesso modo in Banca, Retail o Telecom. Adatta i nomi delle colonne e la logica di business.
Cortex Code ricorda i passi precedenti?
All'interno della stessa sessione di worksheet, sì. Se chiudi e riapri, dovrai fornire il contesto. Per questo raccomandiamo di eseguire tutti i passi di un caso d'uso in una sessione continua.
Posso modificare i prompt?
Sì, ed è consigliato. I prompt sono progettati come punto di partenza. Aggiungi dettagli del tuo business, regola i volumi di dati, cambia soglie o aggiungi colonne specifiche del tuo dominio per ottenere risultati più precisi.
Che differenza c'è tra «Vedi Guida» e «Vedi Prompt»?
Vedi Guida mostra il quadro concettuale: contesto del problema, obiettivi, dati necessari e strategia di implementazione. Vedi Prompt contiene i prompt passo passo che copi e incolli direttamente in Cortex Code per generare il codice.
Devo installare qualcosa?
Quasi nulla. Tutto il codice viene eseguito all'interno di Snowsight nel browser, senza ambienti Python né strumenti esterni. L'unico passo preliminare è scaricare la skill best-practices (file best-practices.zip), decomprimerla sul tuo computer e caricare la cartella in Cortex Code dal pulsante + in Snowsight. Una volta caricata, sarà disponibile con il comando /best-practices e non dovrai ripetere il processo.
